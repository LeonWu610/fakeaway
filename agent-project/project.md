# 模拟外卖项目

## 目标
构建一个移动端优先的模拟点外卖产品：以接近主流外卖产品的信息密度和交互可信度满足“挑选、下单、等待”的情绪需求，同时用原创商家、等待事件和关系成长形成长期留存，并为后续原创内容商业化留出空间。

## 成功标准
- 首页、商家菜单、确认订单、等待配送、情绪结算形成完整且可重复体验的主链路。
- 页面在信息架构、信息密度、布局比例、交互反馈和操作路径上达到真实主流外卖平台级高保真；商标、专有文案、商家素材和品牌识别仍保持原创。
- 商家在头图、菜单结构、优惠、经营状态、内容模块和等待事件上有明显差异。
- 数据支持周期性新增/改写商家和商品；所有待生成图片都可保存结构化 prompt 和审核状态。
- 下单不涉及真实商品、配送或支付，并在关键交易页面明确告知是模拟体验。
- 下单后的等待与送达页提供明确进度、轻互动、个性留言和可积累的情绪收获。

## 关键约束
- 使用子 Agent 时固定选择 `gpt-5.6-sol`（model 82）。
- 保留并核对当前工作区未提交改动，不覆盖用户已有实现。
- 图片阶段先记录结构化生成 prompt；不直接使用真实品牌 Logo、真实人物或受保护角色。
- 产品内使用原创商家和原创营销命名；模拟价格、销量、评价不得使用户误认为真实交易。
- 不接入真实支付；未来付费对象应是原创内容、主题、收藏或功能，而非虚构食品本身。

## 不可破坏项
- 现有 `restaurants.json` 中已经扩充的商家与菜单内容。
- 首页到商家详情页的导航及菜单购物车逻辑。
- 移动端优先、最大宽度 480px 的呈现方式。

## 事实源与验证
- 产品目标与验收反馈：用户当前及后续消息。
- 真实实现：工作区文件和 Git 状态，优先级高于本状态摘要。
- 基础验证：`npm run lint`、`npm run build`，并在移动视口手动走通完整下单链路。
